# Allstar Ecosystem

This repository is documentation-driven.

## Repository structure
- `business/`: business goals and operating context
- `architecture/`: cross-system architecture over all the layers
- `domains/`: bounded domain definitions related with business responsibilities
- `applications/`: frontend application contexts (web-app / mobile-app)
- `services/`: backend service contexts (api-service)
- `infra/`: deployment, networking, CI/CD
- `integrations/`: external system integrations and their corresponding uses.

## Working rules
- Do not change the documentation distribution.
- Respect each file path as part of its meaning.
- Use existing `.md` files as the source of project context.
- Prefer extending current documentation over creating duplicate docs.
